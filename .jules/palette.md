## 2024-05-23 - Custom Component Accessibility\n\n**Learning:** Custom interactive components (like Knobs and Faders) built with `div` elements often completely lack accessibility features, making them invisible or unusable to screen readers and keyboard users. \n\n**Action:** When creating or spotting custom interactive components, always verify: \n1. `role` attribute (e.g., `role="slider"`)\n2. `tabIndex={0}` for focusability\n3. Keyboard event handlers (`onKeyDown`) for interaction\n4. Proper ARIA attributes (`aria-valuenow`, `aria-label`)\n5. `aria-hidden="true"` on redundant visual labels.
